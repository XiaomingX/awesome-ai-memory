# Multi-Agent Coordination & Progress Sync

在多 Agent 协同执行复杂任务（如软件研发、公司运营模拟）时，确保各 Agent 之间的进度、状态和记忆一致性。

## 核心协同框架

| 项目名称 | 协调机制 | 关键应用场景 |
| :--- | :--- | :--- |
| **DeMAC** | 消息队列协调器 | 去中心化协调框架，消除 Zeno 效应，支持 1:1 动态响应。 |
| **Agentic-Sync** | 实时状态共享 | 开发者与 AI 协同，支持每 5s 一次的进度可视化同步。 |
| **Nexus Agents** | Redis A2A 通信 | 研究型多 Agent 系统，实时跟踪进度并内置错误处理机制。 |
| **Consensus MAS** | 事件触发共识 (ETC) | 基于非线性动态系统的学术共识算法，界限事件触发间隔。 |
| **FastAgency** | 生产级测试框架 | 集成 CI/CD 理念，通过 Tester 类验证 Agent 交互的上下文一致性。 |

## 集成建议
1. **状态后端**：建议使用 **Redis** 作为中央共享状态存储，以实现高性能的进度跟踪。
2. **触发策略**：优先采用**事件触发机制 (Event-triggered)** 而非定时轮询，以显著降低通信开销。
3. **规模扩展**：在 Moltbot 多技能协同（如多线小说生成）场景下，需监控共识误差以防止代理偏离主线。
